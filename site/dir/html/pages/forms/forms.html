HTML5 Forms

Traditionally, the term 'form' has referred to a printed document that contains spaces for someone to fill in information. HTML borrows the concept of a form to refer to different elements that allows the manager of a site to collect information from visitors. A form can be something as simple as a search box, or, as complex as an insurance application form. The purpose of HTML5 Forms is to provide a set of elements that facilitates collecting data from users.

Why Forms?

The best known form on the web is probably the search box that sits right in the middle of Google's homepage. In addition to enabling users to search, forms also allow users to perform other functions online such as user registration, checkout purchasing, adding oneself to a newsletter, etc.

How Forms Work

Form Processing

  1. A user begins by filling out a form and then presses a button to submit the information to a server.

  2. The name of each form control is sent to the server along with the value the user enters or selects.

  3. The server processes the information using a programming language such as NodeJS, PHP, C#, VB.net, or Java. It may also store processed information to a database.

  4. The server returns a response based on the information received and processed.

A form may have several form controls, each gathering different information. The server needs to know which piece of inputted data corresponds with which form element.

To differentiate between various pieces of inputted data, information is sent from the browser to the server using the name/value pairs.

Form Structure

Creating A Form

Form controls live inside a <form> Tag. This element should always carry the action attribute and will usually have a method and id attribute set.

  • The action Attribute

    Every <form> element requires an action attribute. Its value is the URL for the page on the server that will receive the information in the form when it is submitted.

  • The method Attribute

    Forms can be sent using one of two methods: GET or POST.

    With the GET Method, the values from the form are added to the end of the URL specified in the action attribute. The GET method is ideal for:

    • Short Forms (Such as search boxes)
    • When the client is retrieving data from the web server. (Not adding or deleting data from the system.)

    With the POST Method, the values from the form are sent in what are known as HTTP headers. As a rule of thumb, The POST method is ideal for:

    • Forms that allow users to upload a file
    • Long Forms
    • Contains sensitive values
    • Adds information to, or deletes information from a database
  • The id Attribute

    The id attribute is used to identify the form distinctly from other elements that might exist on the page. It is often used by scripts - such as those that validation form field values entered in by the user.

  • The DEPRECATED size Attribute

    The size attribute was used on an <input> Tag to define the width of the text input (measured by the number of characters that would be seen). A value of three (3) would create a box wide enough to display three characters (though a user could enter more than three (3) characters). This attribute should not be used in newer versions of code. CSS should be used to control the width of form elements.

If a form is not sent with a method attribute, the form will be sent as GET by default. 

<form
    id="form1"
    action="http://localhost:8080/processForm"
    method="get"
    style="margin: 1rem; padding: 1rem; border: 1px solid black;"
>
  <p>This is where form controls would appear.</p>
</form>

Common Inputs

The <input type="text"/> Tag

When the type attribute has a value of text it creates a text box that acts just like a single-line input field.

The <input type="number"/> Tag

When the type attribute has a value of number it creates an input field that accepts numerical values. Depending on the browser, there the field may include increment/decrement buttons.

The <input type="range"/> Tag

When the type attribute has a value of range defines a control for entering a number whose exact value is not important (like a slider control).
Default range is 0 to 100.

The <input type="email"/> Tag

When the type attribute has a value of email it creates a text input that expects to receive an email address, and may prompt the user as such when the user hovers over the field.

The <input type="password"/> Tag

When the type attribute has a value of password it creates a text box that acts just like a single-line input field, except the characters are blocked out. They are hidden in this way so that if someone is looking over the user's shoulder, they cannot see the sensitive data such as passwords.

The <input type="tel"/> Tag

When the type attribute has a value of tel it creates a text input that expects to receive an telephone number, and may prompt the user as such when the user hovers over the field.

The developer can also specify a pattern attribute by providing a regex validation pattern string to ensure the phone number is in the correct format prior to form submission.
Browsers that do not support "tel" fall back to being a standard "text" input.

The <input type="url"/> Tag

When the type attribute has a value of url it creates a text input that expects to receive a URL, and may prompt the user as such when the user hovers over the field.
The input value is automatically validated before the form can be submitted

The <input type="color"/> Tag

When the type attribute has a value of color it creates a field that will allow the user to select from a palette of colors, or select their own color.

The color value is represented as a hex code value.

Multi-Line Text Input

Creating A TextArea Field

To create a textarea field, specify a set of opening and closing <textarea> Tags. It is used to create a multiline text input.

Unlike other input elements, this is not an empty element: <textarea></textarea>

Any text between the opening and closing <textarea> Tag will be displayed in the textbox when the page loads. If the user does not delete any text between the tags, whatever is in the box will be sent to the server along with what the user typed.

Some sites might implement and use JavaScript to clear this information when the user clicks into the textbox. Which is not out of the box for HTML5.

Radio Buttons & Checkboxes

The <input type="radio"/> Tag

Radio buttons allow users to pick only one (1) of a set of options. The checked attribute can be used to indicate which value (if any) should be selected when the page loads. The value of the attribute is "checked". Only one radio button in a group should use this attribute if provided.

Once a radio button has been selected, it cannot be deselected. The user can only select a different option. If you are only allowing the user one option and want them to be able to deselect it, you should use a checkbox instead.

There are two (2) key factors in choosing between a radio button group and a dropdown list box:

  1. If users need to see all options at a glance, radio buttons are better
  2. If there is a very long list of options such as countries, dropdown list boxes work better.
The <input type="checkbox"/> Tag

Checkboxes allow users to pick between many of a set of options by selecting or deselecting them. Useful for gathering a list of options or determining the functionality of site visits.

Dropdowns & Multi-Select Box Fields

A dropdown list allows users to select one option from a list.

  • The <select> Tag

    The <select> Tag is used to create a dropdown list box. It contains two or more <option> elements.

  • The <option> Tag

    The <option> element is used to specify the options that the user can select from. The words between the opening and closing <option> Tag will be shown to the user in the dropdown.

    • The value Attribute

      The <option> Tag uses the value attribute to indicate the value that is sent to the server along with the name of the control if the option is selected.

    • The selected Attribute

      The <option> Tag uses the selected attribute to set the value of the dropdown field when the page loads. If the selected attribute is not provided, the page will load the first option. If a user does not select an option, the first option will be sent to the server on form submission.

There are two (2) key factors in choosing between a radio button group and a dropdown list box:

  1. If users need to see all options at a glance, radio buttons are better
  2. If there is a very long list of options such as countries, dropdown list boxes work better.

A multiple select box allows users to select more than one option. It contains two or more <option> elements. Unfortunately the way browsers have implemented this attribute is not perfect, and it should be tested thoroughly if used. In particular on FireFox and Safari on MAC.

  • The <select> Tag

    To use a multiple select box, use the implementation of a dropdown list, but add the multiple attribute set to "multiple" to the <select> Tag.

  • The <option> Tag

    Same As Described in the Dropdown <option> Tag Section

File I/O

Developers may want to allow users to provide other formats of data rather than just text or numerical inputs, for instance, Users might be able to upload a photo of themselves for a site avatar. In these instances, developers need to provide the ability to upload files and different file formats. Another example would be when some online form requires someone to upload a Driver's license. To create an upload link, use:

  • The <input type="file"/> Tag
  • Some Available Attributes
    • The id Attribute

      The id attribute is used to identify the field with a unique name.

    • The name Attribute

      The name attribute is used to indicate the name of the form control being sent to the server.

    • The accept Attribute

      The accept attribute is used to restrict the types of files that may be uploaded and sent from the client to the server.

    • The multiple Attribute

      The multiple attribute is used to give the user the ability to upload more than one (1) file at a time.

Developers may also want to provide users the ability to download files and information, for instance, Users might need to be able to download bank statements or reports. To create an download link, use:

  • The <a download/> Tag
  • Specifying the download Attribute

    • The download attribute is used on an <a/> Tag to convert the general behavior of the tag so that rather than routing, it allows the user to receive a file of some type.

Date/Time Inputs

The <input type="date"/> Tag

When the type attribute has a value of date it defines a date control (year, month, day) but no time is included. The label Tag should always be added for accessibility.

The <input type="datetime-local"/> Tag

When the type attribute has a value of datetime-local it defines a date control (year, month, day, time, (no timezone)). The label Tag should always be added for accessibility.

The <input type="time"/> Tag

When the type attribute has a value of time it defines a control for entering a time ( no timezone). The label Tag should always be added for accessibility.

The <input type="month"/> Tag

When the type attribute has a value of month it defines a control for month and year ( no timezone). The label Tag should always be added for accessibility.

The expected field format is: "YYYY-MM"

The <input type="week"/> Tag

When the type attribute has a value of week it defines a week and year control (no time zone). The label Tag should always be added for accessibility.

Some browsers may not display this field correctly.

Buttons & Hidden Controls

The <input type="image"/> Tag

When the type attribute has a value of image it defines an image as a submit button.

A src attribute must be specified for the location of the image.

The <input type="reset"/> Tag

When the type attribute has a value of reset it defines a reset button which resets all form values to its initial values.

Avoid reset buttons in your forms unless a check is performed prior to proceeding with a reset.

The <input type="search"/> Tag

When the type attribute has a value of search it defines a text field for entering a search string.

Remember to set a name for the search field, otherwise nothing will be submitted. The most common name for search inputs is q. And always add a label for accessibility.

The <input type="submit"/> Tag

When the type attribute has a value of submit it defines a submit button which submits all form values to a form-handler.
The form-handler is typically a server page with a script for processing the input data.
The form-handler is specified in the form's action attribute.

Do not use hidden inputs as a form of security!
The <input type="hidden"/> Tag

When the type attribute has a value of hidden it defines a hidden input field.
A hidden field lets web developers include data that cannot be seen by users when a form is submitted.

While the value is not displayed to the user in the page's content, it is visible (and can be edited) using any browser's developer tools or "View Source" functionality.

Labelling & Grouping Form Controls

The <label> Tag

When introducing form controls, the code should have its own <label> Tag as this makes the form accessible to vision impaired users.

<label> Tag can be used in two (2) ways:

  1. Wrap around both the text description and the form input.
  2. Be kept separate from the form control and use the for attribute to indicate which form control it is a label for.
    A for attribute must be specified and set to the id value of the input field.

The for Attribute states which form control the label belongs to. The value of the for attribute matches the id attribute on the form control it is labelling.

As a rule of thumb, the best practices for placing labels on form controls:

Above or to the Left of a Form Control:
  • Text Inputs
  • Text Areas
  • Select Boxes
  • File Uploads
To the Right of a Form Control:
  • Individual Checkboxes
  • Individual Radios
The <fieldset> and <legend> Tags

Related form controls can be grouped together inside the <fieldset> Tag. This is particularly helpful for longer forms.

Most browsers will show the <fieldset> Tag with a line around the edge to show how they are related. The appearance of the lines should be adjusted using CSS.

The <legend> Tag should contain a caption which best describes or identifies the purpose of that group of form controls.

HTML5 Form Validation

Validating HTML5 Form Controls

Traditionally, form validation has been handled by JavaScript. But HTML5 has introduced validation to move the burden of work on the browser. Validation helps ensure the user enters information in a form that the server will be able to understand when the form is submitted. Validation of the contents of the form before it is sent to the server helps:

  • Reduce the amount of work the server has to do
  • Enables users to see if there are any problems with the form (vs validation on the server and something is bad and must be returned to be fixed.)
This does not mean the backend should not perform its own validation. Values can be changed in-flight by hackers post form submission.

Some browser may not support HTML5 validation though they are expected to follow. All use cases should be thoroughly tested. If in doubt, use JavaScript.

Examples:
Implementation
Code
<form action="#formValidation" style="margin: 1rem;" id="formValidation">
  <label for="validateUsername">Username:</label>
  <input id="validateUsername" type="email"
    required="required"
  />
  <label for="validatePassword">Password:</label>
  <input id="validatePassword" type="password"
    required="required"
    minLength="5"
  />
  <input type="submit" value="Try Me!" class="mt-3" />
</form>
