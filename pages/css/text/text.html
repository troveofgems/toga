CSS Text

The properties that allow you to control the appearance of text can be split into two groups:
  1. Those that directly affect the font and its appearance including typefae, whether it is regular, bold, or italic, and the font-size.
  2. Those that would have the same effect on text no matter what font is being used, including the color of the text or the spacing between words and letters.

Formatting text can have a significant effect on how readable pages are.

Typeface Terminology

serif

Serif fonts have extra details on the ends of the main strokes of the letters. These details are known as serifs:

im

In print, serif fonts were traditionally used for long passages of text because they were considered easier to read.

sans-serif

Sans-Serif fonts have straight ends to letters, and therefor have a much cleaner design.

im

Screens have a lower resolution than print. If the text is small, sans-serif fonts can be clearer to read.

monospace

Every letter in a monospace (or fixed-width) font is the same width (Non-monospace fonts have different widths).

Monospace fonts are commonly used for code because they align nicely, making the text easier to follow.

Choosing A Typeface for a Website

How to choose a Typeface?

When choosing a typeface, it is important to understand that a browser will usually only display it if it's installed on that user's computer.

As a result, sites often use a small set of typefaces that are installed on most computers (explore the right panel for more details). There are some techniques to get around this limitation, covered in a section further down.
Go To Section

It is possible to specify more than one typeface and create an order of preference (in case the user does not have the first choice of typefaces installed). This is sometimes referred to as the font stack.

Browsers are supposed to support at least one typeface from each of the groups to the right. For this reason, it is common to add the generic font name after the preferred choice of typefaces.

For example:
font-family: Georgia, Times, serif;

There are several ways to use fonts other than those listed in the prior section. However, typefaces are subject to copyright, so the typefaces used are limited by their respective licenses.

If a site is designed on a Mac, it is important to check what the typefaces look like on PC because PC can render the type less smoothly. But if designed on a PC, it should look fine on Mac.

Specifying Typefaces

Font-Family

The font-family property allows you to specify the typeface that should be used for any text inside the element(s) to which a CSS rule applies.

The value of this property is the name of the typeface that should be used.

A list of fonts may be specified by using a comma to separate each font. The browser will run down the prioritized font list to see if the user has any of them installed in the order you specify.

It is common to end the list with a generic font name for that type of font.

If a font name is made up of more than one word, double quotations should be used to specify the name.

Designers suggest pages usually look better if they use no more than three (3) typefaces on a page.

Type Sizes

Font-Size

The font-size property enables you to specify a size for the font. There are several ways to specify the size of the font. The most common are:

Pixels

Pixels are commonly used because they allow web designers very precise control over how much space their text will take up. The number of pixels is followed by the letters px.

Percentages

The default size of text in browsers is 16px. So a size of 75% would be the equivalent of 12px, and 200% would be 32px.

If a rule is created to make all text inside the <body> element to be 75% of the default size (to make it 12px), and then specify another rule that indicates the content of an element inside the <body> element should be 75%, the resultant size will be 9px (75% of the default 12px).

EM

An em is the equivalent to the width of a letter m.
